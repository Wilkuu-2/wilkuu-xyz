At this point I also feel extremely limited by Processing, which not only limits what API's are exposed to the user, but also makes all your classes inner classes. This causes that most modifiers don't work, for example the visibility modifiers. Another thing that breaks because of that is the static/modifier, which is even worse, since it makes it really confusing to where to put any helper functions that do not change state of any class. The restrictiveness of Processing is felt even more when you start actually writing java with the processing core imported instead. This makes all the classes you write proper classes, instead of being jammed in a jar in form of the transpiled sketch.
